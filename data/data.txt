============================================================
                          INTRO
============================================================
This dataset models store-SKU inventory at period start and
end, daily sales, receiving (purchases), vendor invoices,
and vendor-item purchase prices to enable movement, costing,
and reconciliation workflows.

Core item key: InventoryId (store+SKU)
Purchasing key: VendorNumber + PONumber + InvoiceDate
Vendor-item price reference: Brand + Description + Size + VendorNumber

============================================================
                           KEYS
============================================================
inventory_start: PK = InventoryId
inventory_end: PK = InventoryId
sales: PK = (InventoryId, SalesDate)
purchases: PK = (InventoryId, ReceivingDate, PONumber)
vendor_invoice: PK = (VendorNumber, PONumber, InvoiceDate)
purchase_prices: PK = (Brand, Description, Size, VendorNumber)

============================================================
                      RELATIONSHIPS
============================================================
- sales.InventoryId → inventory_start.InventoryId / inventory_end.InventoryId (M:1)
- purchases.InventoryId → inventory_start.InventoryId / inventory_end.InventoryId (M:1)
- purchases.(VendorNumber,PONumber,InvoiceDate) → vendor_invoice.(VendorNumber,PONumber,InvoiceDate) (M:1)
- purchases.(Brand,Description,Size,VendorNumber) → purchase_prices.(Brand,Description,Size,VendorNumber) (M:1)
- inventory_(start/end).(Brand,Description,Size) → purchase_prices.(Brand,Description,Size) (1:M)

============================================================
                     DERIVATION NOTES
============================================================
- inventory_end = inventory_start + purchases - sales (subject to returns/adjustments)
- purchases lines validated against vendor_invoice headers and purchase_prices
- sales and purchases feed movement analytics; inventory_start/end are period boundaries
